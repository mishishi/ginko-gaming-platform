# 银古客栈 UX 优化方案 C+A：状态感知 + 卡片交互

> 日期：2026-04-22
> 优先级：C（状态感知） > A（卡片交互）

## 一、核心目标

增强游戏卡片的信息层（状态感知）和表现层（交互质感），让用户 **1 秒内扫出可玩游戏**，获得 **明确的触觉/视觉反馈**。

---

## 二、延迟显示增强（C 层核心）

### 默认状态（卡片左上角）

- 小型药丸徽章，显示状态文字 + 颜色
  - 可玩：绿色药丸「可玩 · 32ms」
  - 离线：灰色药丸「离线」
  - 检测中：黄色药丸「检测中」
  - 不可用/维护：红色药丸「不可用」
- 药丸背景 `rounded-full`，字号 `text-xs`，padding `px-2 py-0.5`

### hover 状态（精确值浮层）

- hover 卡片时，药丸旁边展开一个小 tooltip，显示精确延迟值
- tooltip 内容：「延迟 32ms | 更新于 刚刚」
- 150ms 延迟后淡入，避免误触发

### 延迟颜色规范

| 范围 | 颜色 | 色值 |
|------|------|------|
| < 50ms | 绿 | `#22c55e` |
| 50-200ms | 黄 | `#eab308` |
| > 200ms | 红 | `#ef4444` |
| 离线/未知 | 灰 | `#6b7280` |

---

## 三、状态变化通知

### 触发条件

游戏从离线 → 在线，或从不可用 → 可玩

### 动画

1. 状态徽章执行 `pulse-once` 动画（已有）
2. 整卡边框执行一次暖色光晕脉冲（`box-shadow: 0 0 16px rgba(251, 191, 36, 0.4)` → 300ms → 消失）

### 设计决策

- **不打断操作**：无 toast，无 badge，纯视觉引导
- 用户正在浏览卡片列表时，状态变化的卡片自然成为焦点

---

## 四、卡片交互增强（A 层核心）

### hover 效果（桌面端）

| 属性 | 默认值 | hover 值 |
|------|--------|----------|
| 上浮 | `translate-y-0` | `translate-y-1` |
| 边框 | 无 | `box-shadow: 0 0 12px rgba(251, 191, 36, 0.25)` |
| 标题颜色 | 默认 | 轻微变亮 |
| 过渡 | - | `transition: all 200ms ease-out` |

### press 按压效果（点击时）

| 属性 | 值 |
|------|-----|
| scale | `0.95` |
| box-shadow | 轻微内凹 |
| 过渡 | `transition: all 100ms ease-in` |
| 松开后 | 回弹至 scale(1) |

### 收藏按钮动效

- 心形图标：`scale(0.8)` → `scale(1.2)` → `scale(1)`，弹性曲线 300ms
- 光晕：`box-shadow: 0 0 8px rgba(251, 191, 36, 0.5)` 从心形中心扩散，200ms 消失
- 组合：弹跳 + 光晕同时触发

### 移动端点击反馈

- `scale(0.95)` + 150ms ease-out 回弹
- 不同于现有的 0.97（更明确），但保持同一机制

---

## 五、筛选增强

### 新增「只看可玩」快捷 chip

- **位置**：现有筛选按钮右侧
- **样式**：药丸形状，选中时填充绿色
- **行为**：点击后只显示 `status === 'online'` 的游戏

### 筛选过渡动画

- **淡入淡出**：隐藏的卡片 `opacity: 0` + `scale: 0.97`，300ms
- **显示的卡片**：`opacity: 1` + `scale: 1`，300ms
- **交错延迟**：每张卡片间隔 30ms 进入

## 七、设计决策汇总

| 决策点 | 选择 | 原因 |
|--------|------|------|
| 延迟显示方式 | 综合方案（默认药丸 + hover tooltip） | 一眼醒目 + 细节精准 |
| 状态变化通知 | 卡片脉冲闪烁 | 上下文连贯，不打断操作 |
| 发光风格 | 暖色光晕（金/琥珀） | 呼应灯笼符号，古典美学 |
| 收藏动效 | 弹跳 + 光晕组合 | 触觉 + 视觉双重确认 |
| 筛选过渡 | 淡入淡出 300ms | 古典美学，性能友好 |
| 移动端点击反馈 | 缩放弹性 0.95 | 最直接，与现有 press 机制一致 |
