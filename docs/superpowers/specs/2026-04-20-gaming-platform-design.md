# 银古客栈 · 游戏平台 设计文档

## 1. 平台概述

**名称**: 银古客栈
**域名**: openginko.tech
**定位**: 游戏博物馆 + 虫师美学
**愿景**: 用户来到"银古客栈"，像旅人走进幽静驿站，发现三颗各异的光芒（游戏）

---

## 2. 设计语言

### 美学方向
- **平台整体**: 克制的中性美学——深色系、简洁排版、优雅留白，像精心策划的游戏画廊
- **虫师氛围**: 微弱光效、雾气感、自然静谧（灯笼微光、幽绿琥珀色调）
- **游戏卡片**: 各自风格的预览色/图标，平台作为中性画框

### 配色
- **主色**: `#1a1a1a` (深墨色背景)
- **辅色**: `#2d3a2e` (幽绿，虫师自然感)
- **点缀**: `#d4a574` (琥珀/灯笼暖光)
- **文字**: `#e8e4df` (暖白)
- **游戏特色色**:
  - idol: `#ff9ecf` (粉色系，偶像感)
  - quiz: `#00f5ff` (青色，赛博知识感)
  - fate: `#b8945f` (神秘古铜)

### 字体
- 标题: Noto Serif SC（中文衬线，有古典感）
- 正文: Noto Sans SC（简洁易读）

## 4. 页面结构

```
/                           # 首页：游戏大厅
├── /games/idol             # 偶像收集卡牌 (child-game)
├── /games/quiz             # 知识问答竞技 (trivia-card-game)
└── /games/fate             # AI算命占卜 (ai-fortune-telling)
```

### 4.1 首页 (`/`)

**布局**:
- 顶部导航：Logo "银古客栈" + 简约链接
- Hero 区域：平台标题 + 一句话描述（"旅人的游戏驿站"）
- 游戏展品区：三张游戏卡片，横向排列
- 底部：简约 footer

**游戏卡片**:
- 卡片包含：游戏名、简介、一句话描述、代表色
- 悬浮时有微微发光效果（呼应灯笼意象）
- 点击进入对应游戏页面

### 4.2 游戏页面 (`/games/[slug]`)

**布局**:
- 顶部导航：返回首页链接 + 游戏名
- 主区域：iframe 全屏嵌入游戏
- 游戏独立运行，平台仅提供导航壳

**iframe 通信**:
- 各游戏保持独立 window，通过 postMessage 与平台通信
- 平台可发送 theme 指令（iframe 内可选择响应）

| 游戏 | 路由 | 嵌入地址 |
|------|------|----------|
| 偶像收集卡牌 | `/games/idol` | `http://localhost:3002` (dev) |

| AI算命占卜 | `/games/fate` | `http://localhost:3004` (dev) |

> 注：各游戏需分别启动，iframe 嵌入其开发服务器地址。生产环境替换为对应域名。

## 6. 导航栏组件

**设计**:
- 固定顶部，深色半透明背景
- 左：Logo "银古客栈"（可点击返回首页）
- 右：游戏切换下拉（或三个小图标）
- 悬浮效果：虫师微光感

**响应式**:
- 桌面：完整导航
- 移动：汉堡菜单，收起式导航

## 8. 项目初始化

```bash
# 创建 Next.js 项目
npx create-next-app@latest ginko-gaming-platform \
  --typescript \
  --tailwind \
  --app \
  --no-src-dir

# 目录结构
ginko-gaming-platform/
├── app/
│   ├── layout.tsx          # 根布局（含导航栏）
│   ├── page.tsx            # 首页
│   └── games/
│       └── [slug]/
│           └── page.tsx    # 游戏页面（动态路由）
├── components/
│   ├── NavBar.tsx          # 导航栏
│   ├── GameCard.tsx        # 首页游戏卡片
│   └── GameFrame.tsx       # iframe 包装组件
├── docs/
│   └── superpowers/
│       └── specs/          # 设计文档
└── package.json
```
