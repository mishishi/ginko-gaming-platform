# 银古客栈 UX 优化方案

**Goal:** 提升用户体验，增加收藏功能展示，优化移动端布局，增强微交互

**Architecture:** 基于现有组件结构，在首页添加收藏专区，优化 GameCard 交互效果，改进响应式布局

**Tech Stack:** React, Tailwind CSS, localStorage, Next.js App Router

---

## 1. 收藏游戏专区

### 概述
在首页顶部添加"我的收藏"板块，展示用户收藏的游戏卡片，与"最近玩过的"并列显示。

### 组件结构
- 创建 `components/FavoritesSection.tsx`
- 复用现有 `GameCard` 组件
- 在 `app/page.tsx` 中集成

### 数据流
- 使用现有 `useFavorites` hook 获取收藏数据
- 收藏数据从 localStorage 读取
- 筛选出已收藏游戏的完整信息

### 交互
- 收藏为空时显示空状态提示
- 点击游戏卡片跳转游戏页面
- 取消收藏后立即从专区移除（无需刷新）

---

## 2. UI/UX 微交互增强

### 按钮点击反馈
- 所有按钮添加 `active:scale-95` 效果
- 提升点击响应的视觉反馈

### 游戏卡片增强
- Hover: 轻微上浮 + 阴影加深
- Press: `active:scale-98` 缩小效果
- 收藏按钮点击时添加脉冲动画

### 页面过渡
- 复用现有 `PageTransition` 组件
- 确保页面切换流畅

---

## 3. 移动端布局优化

### 响应式网格
- 桌面端: 3-4 列网格
- 平板: 2 列网格
- 手机: 1 列网格

### 导航栏优化
- 移动端收起菜单
- 汉堡菜单展开

### 触摸优化
- 增大触摸目标（按钮、卡片）
- 优化触摸反馈

---

## 验收标准

1. 首页显示"我的收藏"板块（有空状态处理）
2. 收藏游戏点击可跳转
3. 取消收藏后 UI 立即更新
4. 移动端布局正常显示
5. 按钮和卡片的微交互流畅
